## 快速开始

### 前置要求

1. **已编译客户端**  
   客户端应位于 `build/bin/im_client_gui`

2. **IM 服务器运行中**  
   ```bash
   cd ~/Qt_study/my_project
   ./server_im 9999
   ```

3. **网络连接**  
   客户端与服务器应在同一网络或可访问

### 启动客户端

#### 方法 1: 直接运行

```bash
# 设置库路径（如果需要）
export DYLD_LIBRARY_PATH="$HOME/Qt/6.5.3/macos/lib:$DYLD_LIBRARY_PATH"

# 启动客户端
cd ~/Qt_study/my_project/qt_client
./build/bin/im_client_gui
```

#### 方法 2: 使用运行脚本

创建 `run_client.sh`:

```bash
#!/bin/bash
export DYLD_LIBRARY_PATH="${HOME}/Qt/6.5.3/macos/lib:$DYLD_LIBRARY_PATH"
cd "$(dirname "$0")" || exit
./build/bin/im_client_gui "$@"
```

然后：

```bash
chmod +x run_client.sh
./run_client.sh
```

#### 方法 3: 从 macOS 启动台启动（高级）

```bash
# 创建 app 包结构
mkdir -p IMClient.app/Contents/MacOS
mkdir -p IMClient.app/Contents/Resources

# 复制可执行文件
cp build/bin/im_client_gui IMClient.app/Contents/MacOS/

# 创建启动脚本
cat > IMClient.app/Contents/MacOS/launcher.sh << 'EOF'
#!/bin/bash
DIR="$( cd "$( dirname "${BASH_SOURCE[0]}" )" && pwd )"
export DYLD_LIBRARY_PATH="${HOME}/Qt/6.5.3/macos/lib:$DYLD_LIBRARY_PATH"
exec "$DIR/im_client_gui"
EOF

chmod +x IMClient.app/Contents/MacOS/launcher.sh

# 创建 Info.plist
cat > IMClient.app/Contents/Info.plist << 'EOF'
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
    <key>CFBundleName</key>
    <string>IM Client</string>
    <key>CFBundleVersion</key>
    <string>1.0.0</string>
    <key>CFBundleIdentifier</key>
    <string>com.example.imclient</string>
    <key>CFBundleExecutable</key>
    <string>launcher.sh</string>
</dict>
</plist>
EOF

# 双击启动或通过 Spotlight 搜索启动
```

## 登录与认证

### 登录界面

启动客户端后，将显示登录界面：

```
╔══════════════════════════════════════╗
║    IM 客户端 - 登录                  ║
╠══════════════════════════════════════╣
║                                      ║
║  服务器主机: [localhost           ]  ║
║  服务器端口: [9999                ]  ║
║  用户名:    [              ]  ║
║  密码:      [              ]  ║
║                                      ║
║           [登录]      [退出]        ║
║                                      ║
╚══════════════════════════════════════╝
```

### 登录步骤

1. **输入服务器信息**
   - **服务器主机**: IM 服务器的 IP 地址或主机名
     - 本地: `localhost` 或 `127.0.0.1`
     - 远程: `192.168.1.100` 或 `server.example.com`
   - **服务器端口**: 默认 `9999`

2. **输入凭证**
   - **用户名**: 注册时使用的用户名
   - **密码**: 对应的密码

3. **点击 [登录] 按钮**

4. **等待连接**
   - 连接建立中...
   - 认证中...
   - 登录成功！

### 登录常见问题

| 问题 | 原因 | 解决方案 |
|------|------|--------|
| "连接拒绝" | 服务器未运行 | 启动 IM 服务器: `./server_im 9999` |
| "超时" | 网络不通 | 检查防火墙，确保服务器地址/端口正确 |
| "认证失败" | 用户名/密码错误 | 检查凭证，使用正确的用户名和密码 |
| "用户已登录" | 用户在另一设备上已登录 | 等待前一个会话超时或在其他客户端注销 |

### 首次使用（用户注册）

如果还未有用户账户，需要在服务器上创建用户。

通过服务器创建用户：

```bash
# 使用测试工具创建用户
cd ~/Qt_study/my_project

# 运行测试客户端创建用户
./test_codec 127.0.0.1 9999
# 按照提示输入用户信息

# 或查看 README 中的用户创建指导
```

## 单人聊天

### 开始对话

登录后，主界面显示：

#### 1. 从联系人列表选择用户

```
┌─────────────┬──────────────────┐
│   联系人    │                  │
├─────────────┤   聊天记录       │
│ alice   🟢  │                  │
│ bob     🔴  │                  │
│ charlie 🟢  │                  │
│             │                  │
├─────────────┤──────────────────┤
│ + 添加好友  │ 输入消息...      │
│             │ [发送]           │
└─────────────┴──────────────────┘
```

**选择用户**：
- 点击用户名选中用户
- 用户名背景变蓝表示已选中
- 聊天窗口自动切换到与该用户的对话

**用户状态指示**：
- 🟢 **在线**: 用户当前在线，可接收消息
- 🔴 **离线**: 用户已离线（消息可能无法即时送达）

### 发送消息

#### 步骤 1: 选择聊天对象

确保在联系人列表中选中目标用户

#### 步骤 2: 输入消息

在底部消息输入框中输入消息：

```
┌──────────────────────────────────┐
│ 你好，最近怎么样？                  │
└──────────────────────────────────┘
```

#### 步骤 3: 发送

**方式 A: 按 Enter 键**  
```
按 [Enter] 快速发送
```

**方式 B: 点击 [发送] 按钮**  
```
消息输入框右侧的 [发送] 按钮
```

#### 步骤 4: 消息确认

- 消息将显示在聊天窗口中
- 己方消息显示在右侧，蓝色背景
- 对方消息显示在左侧，灰色背景

### 消息格式

$$对话窗口中的消息排版：$$

```
12:34 alice
┌────────────────────────────┐
│ 你好！                      │
└────────────────────────────┘

              12:39 你
         ┌──────────────────┐
         │ 你好，很高兴看到你！   │
         └──────────────────┘

13:40 alice
┌────────────────────────────┐
│ 谢谢！我也很高兴           │
└────────────────────────────┘
```

**元素说明**：
- **时间戳**: 消息发送的时间（hh:mm 格式）
- **发送者**: 消息发送者的用户名
- **消息体**: 实际消息内容

### 消息历史

- **自动保存**: 所有消息在会话中自动保存
- **查看历史**: 在聊天窗口中滚动查看
- **清空历史**: 关闭客户端并重新启动会话时，历史记录会清空（可配置为持久化）

## 群组聊天

### 创建群组

#### 步骤 1: 打开群组管理

在右侧面板的 **"群组"** 选项卡中：

```
┌──────────────────────────────┐
│          群组管理              │
├──────────────────────────────┤
│  我的群组                    │
│ ─ Project Team      (5 人)   │
│ ─ Friends Group     (3 人)   │
│                              │
│  [+ 创建群组]   [退出群组]   │
└──────────────────────────────┘
```

#### 步骤 2: 创建新群组

点击 **[+ 创建群组]**：

```
┌──────────────────────────────┐
│      创建新群组                │
├──────────────────────────────┤
│ 群组名称: [新群组名称      ]   │
│                              │
│  [创建]     [取消]           │
└──────────────────────────────┘
```

输入群组名称，然后点击 **[创建]**

#### 步骤 3: 邀请成员

创建群组后，邀请其他用户加入：

1. 选中群组
2. 在群组信息面板中点击 **[邀请]** 或 **[添加成员]**
3. 选择要邀请的用户
4. 点击 **[确认]**

### 加入群组

#### 自动加入

如果其他用户邀请你加入群组：

1. 你会在聊天窗口中收到加入邀请
2. 选择 **[接受]** 或 **[拒绝]**
3. 接受后，群组自动出现在群组列表中

#### 手动加入

使用群组代码或 ID 加入：

1. 点击 **[加入群组]**
2. 输入群组 ID 或邀请码
3. 点击 **[加入]**

### 群组消息

#### 发送群组消息

1. **选择群组**: 在群组列表中点击群组名称
2. **输入消息**: 在消息框中输入内容
3. **发送**: 按 Enter 或点击 [发送]

#### 群组消息特点

- **广播**: 消息发送给所有群组成员
- **时间戳**: 每条消息都标记时间
- **发送者**: 显示谁发送的消息
- **头像颜色**: 每个成员有不同的头像颜色便于区分

### 群组管理

#### 查看成员列表

选中群组后，右侧面板显示：

```
┌──────────────────────────────┐
│   Project Team (5 人)          │
├──────────────────────────────┤
│ 👤 alice (创建者)             │
│ 👤 bob (成员)                │
│ 👤 charlie (成员)             │
│ 👤 diana (成员)              │
│ 👤 eve (成员)                │
│                              │
│  [邀请] [移除] [退出]        │
└──────────────────────────────┘
```

#### 群组权限

| 角色 | 权限 |
|------|------|
| 创建者 | 创建、邀请、移除成员、解散 |
| 管理员 (可选) | 邀请、移除成员 |
| 成员 | 发送消息、查看历史 |

#### 管理操作

- **邀请成员**: [邀请] 按钮 → 选择用户 → [确认]
- **移除成员**: 选中成员 → [移除] → [确认]
- **退出群组**: [退出] 按钮 → [确认]
- **解散群组** (创建者): [更多] → [解散群组] → [确认]

## 功能详解

### 消息驱动架构

客户端与服务器之间的所有通信都基于二进制消息协议：

$$\text{HTTP} \rightarrow \text{Binary Protocol}$$

**消息结构**:

```
┌──────┬────────┬────────┬──────────┐
│ Magic│  Type  │ Length │   Body   │
│ 4B   │ 2B     │ 4B     │ Variable │
├──────┼────────┼────────┼──────────┤
│ 0xdead │ 110  │ 48     │ {"..."}  │
└──────┴────────┴────────┴──────────┘
```

### 支持的消息类型

| 类型 | 值 | 方向 | 说明 |
|------|-----|-----|------|
| LOGIN | 101 | C→S | 用户登录 |
| LOGOUT | 102 | C→S | 用户登出 |
| CHAT_TO | 110 | C↔S | 单人消息 |
| CHAT_GROUP | 111 | C↔S | 群组消息 |
| CREATE_GROUP | 120 | C→S | 创建群组 |
| JOIN_GROUP | 121 | C→S | 加入群组 |
| LIST_GROUPS | 122 | C→S | 查询群组列表 |
| GROUP_MEMBERS | 123 | C→S | 查询群组成员 |
| NOTIFY | 130 | S→C | 服务器通知 |
| ACK | 200 | S→C | 消息确认 |

### 线程模型

客户端使用 **多线程 + Signal/Slot** 架构：

```
┌────────────────────────┐
│   GUI 线程 (主线程)     │
│ - UI 更新               │
│ - 事件处理             │
│ - Signal/Slot dispatch │
└────────────────────────┘
         ↕
┌────────────────────────┐
│ 网络线程 (QThread)     │
│ - TCP socket 管理      │
│ - 消息编码/解码        │
│ - 异步网络 I/O        │
└────────────────────────┘
```

**优点**:
- 网络操作不阻止 UI
- 流畅的用户交互
- 低延迟的消息处理

### 自动消继续

客户端支持自动心跳保活：

```bash
# 每 30 秒发送一次心跳包
HEARTBEAT_INTERVAL = 30s

if time_since_last_activity > 30s:
    send_heartbeat()
    last_activity = now()
```

**目的**:
- 保持连接活跃
- 检测连接状态
- 防止服务器超时

## 故障排查

### ❌ 问题 1: "连接拒绝"

**症状**: 
```
[错误] 连接被拒绝: Connection refused (127.0.0.1:9999)
```

**原因**: IM 服务器未运行

**解决方案**:

```bash
# 启动 IM 服务器
cd ~/Qt_study/my_project
./server_im 9999

# 检查服务器状态
netstat -an | grep 9999
# 应显示: tcp4  0  0  127.0.0.1.9999 ...LISTEN
```

### ❌ 问题 2: "认证失败"

**症状**:
```
[错误] 登录失败: 用户名或密码错误
```

**原因**: 凭证不匹配

**解决方案**:

```bash
# 1. 验证用户是否存在
   # 使用测试工具查询用户信息

# 2. 重置密码（需要服务器支持）
   # 或创建新用户账户

# 3. 检查用户名是否大小写敏感
   # 大多数系统区分大小写
```

### ❌ 问题 3: "消息发送失败"

**症状**:
```
[错误] 发送消息时出错: 网络错误
```

**原因**: 网络链接断开或消息过大

**解决方案**:

```bash
# 1. 检查网络连接
ping 127.0.0.1

# 2. 检查消息长度
# 消息长度限制: 通常为 64KB

# 3. 重新连接
# 使用菜单: 文件 → 重新连接

# 4. 查看详细错误
# 启用调试模式查看完整日志
```

### ❌ 问题 4: "无法找到 libQt6Core.so"

**症状**:
```
dyld: Library not loaded: @rpath/libQt6Core.framework
Reason: image not found
```

**原因**: 运行时库路径未配置

**解决方案**:

```bash
# 设置库路径环境变量
export DYLD_LIBRARY_PATH="/Users/username/Qt/6.5.3/macos/lib:$DYLD_LIBRARY_PATH"

# 验证库存在
ls -la /Users/username/Qt/6.5.3/macos/lib/ | grep Core

# 运行客户端
./build/bin/im_client_gui
```

### ❌ 问题 5: "客户端崩溃或冻结"

**症状**: 应用无响应或意外退出

**解决方案**:

```bash
# 1. 查看系统日志
log stream --level debug | grep im_client

# 2. 以调试模式运行
lldb ./build/bin/im_client_gui
(lldb) run
# ... 应用崩溃时显示堆栈跟踪

# 3. 检查内存使用
top -p $(pgrep im_client_gui)

# 4. 强制杀死进程（如卡死）
killall im_client_gui
```

### ❌ 问题 6: "群组消息显示混乱"

**症状**: 消息时间戳或发送者不正确

**解决方案**:

```bash
# 1. 清空应用缓存
# 删除本地消息历史（如有）

# 2. 重新登录
# 登出后重新登入

# 3. 重启服务器和客户端
./server_im 9999  # 新终端
./build/bin/im_client_gui  # 新终端
```

### ❌ 问题 7: "显示缩放问题（Retina 屏幕）"

**症状**: UI 在高 DPI 屏幕上显示不正确

**解决方案**:

